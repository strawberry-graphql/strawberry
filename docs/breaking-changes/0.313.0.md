# v0.313.0 Breaking Changes

This release moves connection pagination types from `strawberry.relay` into a
new `strawberry.pagination` module and renames the `relay_max_results` config
option.

## Deprecated: Pagination imports from `strawberry.relay`

`Connection`, `ListConnection`, `Edge`, `PageInfo`, `connection`,
`ConnectionExtension`, `from_base64`, and `to_base64` should now be imported
from `strawberry.pagination`:

```python
# Before
from strawberry.relay import Connection, ListConnection, connection

# After
from strawberry.pagination import Connection, ListConnection, connection
```

Importing from `strawberry.relay` still works but will emit a
`DeprecationWarning`.

## Deprecated: `relay_max_results` config option

The `relay_max_results` option in `StrawberryConfig` has been renamed to
`connection_max_results`:

```python
# Before
schema = strawberry.Schema(query=Query, config=StrawberryConfig(relay_max_results=50))

# After
schema = strawberry.Schema(
    query=Query, config=StrawberryConfig(connection_max_results=50)
)
```

The old name is still accepted but will emit a `DeprecationWarning`.

## Codemod

A codemod is available to automatically update your code:

```bash
strawberry upgrade update-relay-imports .
```

This will update both imports and the config option name.
